\section{Pendahuluan}

BTC-QUANT adalah sistem scalping algoritmik berbasis model sinyal multi-layer
(HMM + BCD + MLP) yang dieksekusi pada pasar perpetual futures BTC/USDC di
Lighter.xyz---sebuah ZK-rollup DEX dengan biaya taker/maker 0\%.

Laporan ini mencakup tiga fase utama: (i)~infrastruktur awal dan first live order,
(ii)~identifikasi dan perbaikan bug kritikal pada execution layer, dan
(iii)~optimasi parameter strategi berdasarkan data empiris.

\section{Fase 1: Infrastruktur Awal (10--16 Mar)}

Deployment awal mencakup integrasi SDK \texttt{lighter-python}, setup Docker container
(\texttt{btc-quant-api}) pada VPS, dan konfigurasi environment variables.

\section{Fase 2: Bug Fixes Kritikal (17--25 Mar)}

\subsection{Double Position Guard \fixlabel{FIX-1}}
\textbf{Commit:} \texttt{492cf0b}

Bot dapat membuka posisi baru ketika posisi lama masih OPEN, menyebabkan double
exposure. Fix: validasi \texttt{get\_open\_position()} sebelum setiap entry attempt.

\subsection{Signal Replay Prevention \fixlabel{FIX-2}}
\textbf{Commit:} \texttt{d8e7a87}

\texttt{last\_notified\_ts} hilang dari memory saat restart, memungkinkan sinyal lama
di-\textit{replay}. Fix: persist ke disk (\texttt{last\_candle\_ts.json}).

\subsection{SL Order Authentication \fixlabel{FIX-3}}
\textbf{Commit:} \texttt{c3107f9}

SL order gagal ditempatkan akibat auth header yang tidak benar pada endpoint order
placement. Ini adalah penyebab langsung anomali trade 18 Maret (-\$6.30) di mana
posisi tidak terlindungi SL.

\subsection{Advisory Signal Gate \fixlabel{FIX-4}}
\textbf{Commit:} \texttt{3160dc1}

Signal dengan conviction 15--20\% (status ADVISORY) di-skip oleh gate logic.
Fix: ADVISORY signals diperlakukan ekuivalen dengan ACTIVE untuk entry.

\subsection{SDK Auth Token \fixlabel{FIX-5}}
\textbf{Commit:} \texttt{e28c312}

Auth token dibuat secara manual dengan format \texttt{\{expiry\}:\{account\}:\{key\}:\{hex\}}
yang tidak valid---Lighter API membutuhkan token bertanda tangan kriptografis.

\begin{lstlisting}[language=Python, caption=Auth Token Fix]
# Broken: manual format (unsigned)
token = f"{expiry}:{account}:{key}:{hex}"

# Fixed: SDK signed token
result = client.create_auth_token_with_expiry(
    deadline=SignerClient.DEFAULT_10_MIN_AUTH_EXPIRY,
    api_key_index=self.api_key_index,
)
token = result[0]
\end{lstlisting}

\subsection{Exit Price Detection \fixlabel{FIX-6}}
\textbf{Commit:} \texttt{e28c312}

\texttt{fetch\_last\_closed\_order()} return \texttt{filled\_price = 0.0} karena
Lighter SL/TP adalah \textit{reduce-only trigger order}---harga eksekusi tersimpan
di field \texttt{trigger\_price}, bukan \texttt{filled\_base\_amount}.

\begin{lstlisting}[language=Python, caption=Exit Price Fix]
filled_base = float(order.get(
    "filled_base_amount", 0) or 0)
if filled_base > 0:
    price = filled_quote / filled_base
else:
    price = float(order.get(
        "trigger_price", 0) or 0)
\end{lstlisting}

\subsection{Startup Position Sync \fixlabel{FIX-7}}
\textbf{Commit:} \texttt{861822f}

Restart saat posisi open menyebabkan desinkronisasi DB dan exchange.
Fix: eksekusi \texttt{sync\_position\_status()} saat daemon start.

\subsection{Exit Type dari order\_type \fixlabel{FIX-8}}
\textbf{Commit:} \texttt{861822f}

Exit type (SL/TP) ditentukan dari jarak harga---tidak reliable.
Fix: gunakan field \texttt{order\_type} dari API response.

\subsection{L2 EMA Weakening \fixlabel{FIX-9}}
\textbf{Commit:} \texttt{8ab9b1b}

L2 EMA confirmation hanya mengurangi confidence, tidak memblokir sepenuhnya.
Fix: force \texttt{l2 = False} saat kondisi weakening aktif.

\subsection{OCO Canceled Order Fix \fixlabel{FIX-10}}
\textbf{Commit:} \texttt{f644601}

\textbf{Root cause kritikal:} Ketika TP hit, SL companion di-cancel otomatis (OCO
behavior). Order canceled ini memiliki \texttt{trigger\_price} valid sehingga
bot salah mengklasifikasikan exit sebagai SL, mengakibatkan SL freeze palsu
dan memblokir entry berikutnya.

\begin{lstlisting}[language=Python, caption=OCO Fix]
for order in closed_orders:
    status = order.get("status", "")
    # Skip auto-canceled OCO companion
    if "cancel" in status.lower():
        continue
    # Process actually filled order
    ...
\end{lstlisting}

\subsection{Interpretasi}

Meskipun R:R $<$ 1 di semua fase---mengindikasikan rata-rata win lebih kecil dari
loss---sistem tetap profitable karena win rate tinggi (73--80\%) mengkompensasi
asymmetry tersebut. Hubungan ini dinyatakan oleh:

\begin{equation}
\text{EV} = p_w \cdot \bar{W} + (1 - p_w) \cdot \bar{L} > 0
\end{equation}

Di mana $p_w = 0.731$, $\bar{W} = +\$0.501$, $\bar{L} = -\$0.670$
menghasilkan EV $= +\$0.186$ per trade.

Profit Factor 2.03 menunjukkan sistem menghasilkan \$2.03 untuk setiap \$1
kerugian---threshold \textit{acceptable} untuk strategi intraday.

\small\textbf{Catatan anomali:} Trade 18 Mar (-\$6.30) setara dengan $\sim$13

\section{Kesimpulan}

Sistem BTC-QUANT berhasil bertransisi ke live trading otomatis di Lighter mainnet
dengan 12 perbaikan teknis yang meningkatkan keandalan eksekusi secara signifikan.
Fase otomatis menunjukkan performa superior dibanding fase manual (WR 80\% vs 63.6\%,
PF 3.53 vs 1.22), mengkonfirmasi validitas arsitektur sinyal multi-layer.

Satu-satunya hambatan signifikan adalah anomali SL Bug (18 Mar) yang telah
diperbaiki---tanpa anomali tersebut, sistem dalam kondisi profitable dengan margin
yang substansial. Dengan update parameter ke \$20$\times$7x, profit per trade
diproyeksikan naik $\approx$1.87$\times$ dengan risiko likuidasi yang tetap terkontrol.
